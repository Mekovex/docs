Лабораторная работа: Анализ журнала событий

1. Цель работы
Научиться читать журнал событий осмысленно, находить в логе ключевые события перед отказом, восстанавливать последовательность происходившего, формулировать гипотезу о причине сбоя, отличать симптом от первопричины и делать профессиональный вывод по инциденту.

2. Описание инцидента
**Название проекта:** «Сервис задач» (консольный модуль управления задачами)

**Дата анализа:** 25 июня 2026

**Краткое описание проблемы:**
Пользователь сообщает: «Программа вчера работала, я создал задачи, сохранил их, а сегодня после запуска приложение падает при загрузке данных».

**Источник данных:** Учебный фрагмент журнала событий (предоставлен в задании)

3. Фрагмент журнала событий
```
2026-03-11 10:14:01 INFO app: application started
2026-03-11 10:14:01 INFO storage: loading data from data/tasks.json
2026-03-11 10:14:01 INFO storage: file found, size=248 bytes
2026-03-11 10:14:01 ERROR storage: failed to parse json
2026-03-11 10:14:01 ERROR storage: exception=JSONDecodeError: Expecting ',' delimiter: line 1 column 119 (char 118)

2026-03-11 10:14:01 ERROR app: startup aborted
2026-03-11 10:14:01 INFO app: application finished with code 1
```

4. Ключевые строки журнала
1. `INFO app: application started` — программа запущена
2. `INFO storage: loading data from data/tasks.json` — начата загрузка файла данных
3. `INFO storage: file found, size=248 bytes` — файл найден, имеет размер 248 байт
4. `ERROR storage: failed to parse json` — ошибка при разборе JSON
5. `ERROR storage: exception=JSONDecodeError: Expecting ',' delimiter: line 1 column 119 (char 118)` — техническая ошибка: не хватает запятой
6. `WARNING app: fallback to empty task list was not applied` — резервный сценарий не сработал
7. `ERROR app: startup aborted` — запуск прерван
8. `INFO app: application finished with code 1` — приложение завершено с кодом ошибки

5. Последовательность событий
1. Программа была запущена.
2. Модуль хранения начал загрузку файла `data/tasks.json`.
3. Файл был найден и имел размер 248 байт (не пустой).
4. При попытке разбора содержимого возникла ошибка JSONDecodeError.
5. Программа не смогла перейти к безопасному сценарию (пустой список задач).
6. Запуск был прерван.
7. Приложение завершилось с кодом ошибки 1.

6. Симптом и возможные причины

**Симптом:**
Программа падает при запуске, данные не загружаются.

**Техническое проявление симптома:**
Ошибка `JSONDecodeError: Expecting ',' delimiter: line 1 column 119 (char 118)`.

**Возможные причины:**
1. Файл `data/tasks.json` повреждён.
2. Файл был записан не полностью (обрыв записи).
3. В файл попали недопустимые символы.
4. Формат файла изменился, а загрузчик остался старым.
5. Логика обработки ошибок не поддерживает аварийное восстановление.
6. Резервный сценарий (переход к пустому списку) отсутствует или неисправен.

7. Гипотеза о первопричине
Наиболее вероятная первопричина состоит в том, что файл `data/tasks.json` содержит некорректный JSON и не может быть разобран. На это указывает сообщение `JSONDecodeError`, а также тот факт, что файл был найден и прочитан, но разбор структуры завершился ошибкой. Дополнительной причиной отказа запуска является отсутствие или неисправность механизма безопасного перехода к пустому списку задач после ошибки чтения.

8. Обоснование гипотезы по строкам лога
1. Строка `file found, size=248 bytes` показывает, что проблема не в отсутствии файла — файл существует и имеет ненулевой размер.
2. Строка `failed to parse json` показывает, что проблема именно в содержимом файла, а не в его наличии или доступе к нему.
3. Строка `JSONDecodeError: Expecting ',' delimiter: line 1 column 119` указывает на конкретное место ошибки — не хватает запятой между элементами JSON.
4. Строка `fallback to empty task list was not applied` показывает, что сбой стал критическим из-за отсутствия безопасной обработки ошибки — программа не переключилась на пустой список задач.
5. Строка `startup aborted` показывает результат этой цепочки — запуск прерван.

9. Вывод по первопричине
1. **Что произошло:** При попытке загрузки данных из файла `data/tasks.json` возникла ошибка разбора JSON из-за того, что файл содержит некорректную структуру (отсутствует запятая).
2. **Почему произошло:** Файл данных был повреждён или записан с ошибкой. Возможно, это произошло из-за аварийного завершения программы во время сохранения, либо из-за ручного редактирования файла.
3. **Почему программа завершилась аварийно:** Программа не обработала ошибку чтения корректно — резервный сценарий перехода к пустому списку задач (`fallback to empty task list`) не был применён, поэтому запуск был полностью прерван.

**Вывод:** Первопричиной инцидента является некорректное содержимое файла данных `tasks.json`, которое не соответствует формату JSON. Ошибка разбора привела к невозможности загрузить задачи. Критичность инцидента усилилась тем, что приложение не применило резервный сценарий перехода к пустому списку задач, поэтому запуск был прерван полностью.

10. Предложение по исправлению
Для устранения проблемы необходимо:
1. Добавить проверку корректности JSON перед использованием (валидация структуры).
2. Реализовать безопасный переход к пустому списку задач при ошибке чтения (обработка исключения с переходом в безопасное состояние).
3. Логировать путь к файлу и содержимое ошибки более подробно (добавить в лог путь к файлу и строку с ошибкой).
4. Защитить запись файла от частичного сохранения (использовать временный файл и переименовывать его после успешной записи).

11. Предложение по проверке, предотвращающей повторение
После исправления необходимо добавить регрессионную проверку:
- Если файл JSON повреждён, программа не должна аварийно завершаться, а должна выводить диагностическое сообщение и переходить к пустому списку задач.
- Добавить модульный тест: запуск программы при повреждённом файле `tasks.json` должен завершаться успешно с пустым списком задач.
- Добавить тест на запуск программы при отсутствующем файле данных (должна создаваться новая пустая база задач без ошибок).
